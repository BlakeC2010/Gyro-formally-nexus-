# GitHub Copilot Chat Setup for Gyro

This document guides you through setting up Copilot Chat access from any device.

## Quick Start (Any Device)

### 1️⃣ Clone the Repository
```bash
git clone https://github.com/BlakeC2010/nexus.git
cd nexus
```

### 2️⃣ Set Up Python Environment

**Windows:**
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

**Mac/Linux:**
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3️⃣ Install VS Code Extensions
In VS Code, install these extensions:
- **GitHub Copilot** (Id: `GitHub.copilot`)
- **GitHub Copilot Chat** (Id: `GitHub.copilot-chat`)
- **Python** (Id: `ms-python.python`) — optional but recommended

Install from the Extensions marketplace or run:
```bash
code --install-extension GitHub.copilot GitHub.copilot-chat ms-python.python
```

### 4️⃣ Sign In to Copilot
1. Press `Ctrl+Shift+P` (or `Cmd+Shift+P` on Mac)
2. Type "GitHub Copilot: Sign in"
3. Follow the browser dialog to authenticate with your GitHub account
4. You're done! Copilot Chat is now available.

### 5️⃣ Enable Settings Sync (Optional)
Keep your VS Code settings and extensions synced across all devices:
1. Press `Ctrl+Shift+P`
2. Type "Settings Sync: Turn On"
3. Sign in with GitHub
4. On other devices, enable Settings Sync — it auto-restores everything

## Using Copilot Chat

Once set up, you can:

- **Chat**: Open the Copilot Chat panel (`Ctrl+I` or from the activity bar)
- **Ask questions** about the codebase
- **Request edits**: Select code and use Copilot to refactor/modify
- **Generate code**: Describe what you want, Copilot creates it
- **Debug**: Paste errors and ask for fixes
- **Document**: Ask Copilot to write docstrings, READMEs, etc.

## Troubleshooting

**Copilot Chat not appearing?**
- Restart VS Code
- Ensure you're signed in: `Ctrl+Shift+P` → "GitHub Copilot: Sign in"
- Check your GitHub account has a Copilot subscription

**Python venv not activating?**
- Windows: Make sure PowerShell execution policy allows scripts. Run as admin:
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```
- Mac/Linux: Ensure you're in the right shell (`bash`, `zsh`, etc.)

**Dependencies won't install?**
- Upgrade pip: `python -m pip install --upgrade pip`
- Try: `pip install -r requirements.txt --no-cache-dir`
